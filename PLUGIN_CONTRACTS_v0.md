# PLUGIN_CONTRACTS_v0

Objective: freeze internal plugin contracts for FORGE extensibility.

Scope: **interfaces only** (no new providers/targets/rulesets implemented here).

---

## AdvisoryProvider v0

### Interface

- Location: `forge/src/policy/advisories/provider.ts`

```ts
export type AdvisoryProviderContext = {
  environment?: {
    nodeVersion?: string;
    platform?: string;
  };
};

export interface AdvisoryProvider {
  readonly id: string;
  getAdvisories(input: { spec: Spec; ctx: AdvisoryProviderContext }): Promise<Advisory[]> | Advisory[];
}
```

### Input / Output

- Input:
  - `spec`: full Spec object (already parsed)
  - `ctx`: environment metadata (best-effort; may be missing)
- Output:
  - `Advisory[]`

### Advisory (required fields)

Every advisory **must** include:
- `id` (string)
- `package` (string)
- `severity` (`low | moderate | high | critical`)
- `title` (string)
- `range_affected` (string)
- `recommendation` (string)
- `source` (string)
- `url` (string, optional)

### Rules

- `source` is **required**.
- `source` initial enum set:
  - `seeded`
  - `npm_audit`
  - `custom`

(Additional sources may be added later, but existing values must not change.)

## PolicyRuleset v0

### Interface

- Location: `forge/src/policy/ruleset.ts`

```ts
export type GovernanceConfig = {
  enforcement_mode: EnforcementMode;
};

export type PolicyEvaluation = {
  ok: boolean;
  errors: string[];
  warnings: string[];
  advisories: Advisory[];
  deployBlockers: string[];
  enforcementMode: EnforcementMode;
};

export interface PolicyRuleset {
  readonly id: string;
  evaluate(input: { spec: Spec; governance: GovernanceConfig; advisories: Advisory[] }): PolicyEvaluation;
}
```

### PolicyEvaluation schema (contract)

- Gate outcome:
  - `ok: boolean` (PASS if true; FAIL if false)
  - `errors: string[]` (human-readable; empty when ok)
- Messages:
  - `warnings: string[]` (human-readable)
- Deployment control:
  - `deployBlockers: string[]` (human-readable; may be empty)

### Rule: blockers require reason_code

**Contract requirement (v0):** every deploy blocker must carry a `reason_code` (short string).

Current runtime representation uses strings for blockers; before allowing third-party rulesets, the internal representation must evolve to include a structured blocker object:

```ts
type DeployBlocker = { reason_code: string; message: string };
```

Until that evolution lands, rulesets must generate blocker strings in the format:

```
<reason_code>: <message>
```

Where `<reason_code>` is a short stable token (e.g. `SEC_ADVISORY_HIGH`).

## CompilerTarget v0

### Interface

- Location: `forge/src/compiler/targets/target.ts`

```ts
export type TargetContext = {
  specHash: string;
  schemaVersion: string;
  buildTime: string;
};

export type GeneratedArtifact = {
  relPath: string;
  content: string;
};

export interface CompilerTarget {
  readonly id: string;
  rootDirName(): string;
  generate(input: { spec: Spec; appName: string; ctx: TargetContext }): Promise<GeneratedArtifact[]> | GeneratedArtifact[];
}
```

### GeneratedArtifact[] (contract)

Each artifact must include:
- `path` (relative path)
- `type` (file type / mime-like discriminator)
- `checksum` (optional)

**Note:** current implementation uses `{ relPath, content }`. Before accepting external targets, the artifact model must be extended to:

```ts
type GeneratedArtifact = {
  path: string;
  type: 'text';
  content: string;
  checksum?: string;
};
```

### Rule: standard header required

All generated source files must receive the standard header containing:
- `SPEC_HASH`
- `SCHEMA_VERSION`

The exact header format is currently:

```
/**
 * GENERATED BY FORGE — DO NOT EDIT MANUALLY
 * SPEC_HASH: <sha256>
 * SCHEMA_VERSION: <v0>
 */
```

Targets must apply it consistently to all generated files that support headers (TS/TSX/etc).
